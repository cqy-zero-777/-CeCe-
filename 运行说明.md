# AI测试用例生成工具 - 小Q V4.0

## 🚀 快速启动

### 方式一：双击运行（推荐）
双击项目根目录下的 `run.bat` 文件

### 方式二：命令行运行
```bash
cd d:\demo\demo
mvn.cmd spring-boot:run
```

### 方式三：使用JAR包运行
```bash
cd d:\demo\demo
mvn.cmd clean package -DskipTests
java -jar target\demo-0.0.1-SNAPSHOT.jar
```

## 📍 访问地址

启动成功后，访问：
- **主页面**: http://localhost:8886
- **Swagger文档**: http://localhost:8886/swagger-ui.html

## 🎯 功能模块

### Tab1: 从接口文档生成用例
- 支持上传 .docx、.json、.yaml、.yml 格式的接口文档
- 支持读取范围选择（全部读取/指定页数/指定章节）
- 大型文档推荐使用"指定页数"或"指定章节"功能
- 自动提取接口并生成测试用例

### Tab2: 从需求文档生成用例
- 支持上传需求文档
- 可选择文档类型（接口文档/需求文档）
- 支持读取范围选择
- 自动识别业务流程并生成功能测试用例

### Tab3: 生成自动化脚本
- 基于接口文档生成 TestNG + RestAssured + Allure 自动化测试脚本
- 支持自定义 BaseURL
- 自动打包为 ZIP 文件下载

### Tab4: 文件上传
- 快速上传接口文档
- 自动解析并生成测试用例

### Tab5: URL方式
- 输入 Swagger/OpenAPI 文档的 URL 地址
- 自动解析远程文档

### Tab6: 自然语言
- 输入接口描述或需求描述
- 支持自定义测试要求
- 可选择文档类型（接口文档/需求文档）

### Tab7: 批量扫描
- 批量处理文件夹中的所有文档
- 生成统一的测试用例报告

## ⚙️ 配置说明

### API Key 配置
在使用 AI 功能前，请配置您的 DeepSeek API Key：

**方式1：环境变量**
```bash
set DEEPSEEK_API_KEY=您的API密钥
```

**方式2：修改 application.yml**
```yaml
ai:
  api-key: 您的API密钥
```

### 端口配置
默认端口：**8886**

如需修改，编辑 `src/main/resources/application.yml`：
```yaml
server:
  port: 8886  # 修改为您需要的端口
```

## 📊 生成的测试用例格式

| 字段 | 说明 |
|------|------|
| 用例编号 | 模块名+编号（如：LOGIN-001） |
| 测试模块 | 根据接口路径自动识别 |
| 测试点 | 如：正常登录、参数校验、边界值 |
| 前置条件 | 测试前需要满足的条件 |
| 具体操作 | 详细的测试步骤 |
| 请求参数 | 接口参数 |
| 预期结果 | 期望的系统响应 |
| 优先级 | 高/中/低 |

## 📝 常见问题

### 1. 启动失败，端口被占用
修改 `application.yml` 中的 `server.port` 为其他端口

### 2. AI 功能不可用
检查 API Key 是否正确配置

### 3. 文档解析失败
确保文档格式正确，包含有效的接口或需求描述

### 4. 大型文档处理慢
使用"读取范围"功能，只处理需要的部分

## 🎨 技术栈

- **后端**: Spring Boot 3.3.0
- **前端**: 原生 HTML/CSS/JavaScript
- **AI服务**: LangChain4j + DeepSeek API
- **文档解析**: Apache POI (Word)、Swagger Parser
- **自动化脚本**: TestNG + RestAssured + Allure
